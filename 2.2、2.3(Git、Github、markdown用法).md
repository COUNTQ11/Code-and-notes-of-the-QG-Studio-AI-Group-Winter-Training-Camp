# **一．Git以及Github**

1. Git的三个概念：commit(提交，每一次提交都会被保存下来)、repository(仓库，即存储项目的地方)、branch(分支，即根据不同人或需求开发不同的功能最后再整合)。

2. **Git操作路径**

   ### 第1步：安装与基础配置

   ```
   1# 安装Git后，设置你的身份信息
   2git config --global user.name "你的名字"
   3git config --global user.email "你的邮箱@example.com"
   ```

   ### 第2步：理解核心概念

   - **仓库(Repository)**: 存储项目所有文件和历史记录的地方
   - **提交(Commit)**: 保存项目在某一时刻的快照
   - **分支(Branch)**: 独立的开发线，让你可以在不影响主代码的情况下开发新功能
   - **远程仓库**: 托管在GitHub、GitLab等平台上的共享仓库

   ### 第3步：基本工作流程

   1. **初始化仓库**：`git init`
   2. **添加文件到暂存区**：`git add .`
   3. **提交更改**：`git commit -m "描述你的更改"`
   4. **查看状态**：`git status`
   5. **查看历史**：`git log`

   ### 第4步：分支操作

   ```
   # 1.创建新分支
   git branch feature-name
   
   # 2.切换到分支
   git checkout feature-name
   
   # 3.创建并切换到新分支（常用）
   git checkout -b feature-name
   
   # 4.合并分支
   git merge feature-name
   ```

   ### 第5步：与远程仓库协作

   ```
   # 1.克隆远程仓库
   git clone https://github.com/用户名/仓库名.git
   
   # 2.推送本地更改到远程
   git push origin branch-name
   
   # 3.从远程获取更新
   git pull origin branch-name
   ```

5. Stars类似于收藏数，在搜索时可以在搜索内容后加上stars此时会有数字的选项，也可以自己手动输入stars的数量要求。

##   **`提示`**

1. **换行技巧**：
   - 段落间：空一行
   
   - 行内换行：行尾加 **两个空格** + 回车（GitHub 标准）
   
2. **图片管理**：
   在笔记同级建 `images/` 文件夹 → 上传图片 → 用 `![描述](images/xxx.png)` 引用（仓库迁移也不怕丢图！）

3. **实时预览**：
   GitHub 编辑 `.md` 文件时右侧有 **Preview** 标签，边写边看效果

4. **转义字符**：
   想显示 `*` 本身？写成 `\*` 即可

5. **调整字体大小**：

   - 增大字体：Ctrl + "+" (Windows)

   - 减小字体：Ctrl + "-" (Windows)
